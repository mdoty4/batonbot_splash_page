## 4. How It Works (3-Step Flow)

### Section Heading

> **From Idea to Output in Three Steps**

### Step 1 — Describe

**Step Label:** `01`

**Title:** Describe the Work

**Body:** Tell Batonbot what you need. A feature, a refactor, a set of tests — describe it in your own words. Batonbot breaks it into actionable steps and presents them in a clear, ordered flow.

---

### Step 2 — Arrange

**Step Label:** `02`

**Title:** Arrange the Order

**Body:** Reorder steps. Split complex tasks. Assign specific agents to specific stages. You control the sequence — because the order in which work happens matters. Drag, drop, and arrange until the flow is right.

---

### Step 3 — Execute

**Step Label:** `03`

**Title:** Execute & Review

**Body:** Hit run. Batonbot's agents execute each step in order, passing results along the chain. Track progress at each stage. Inspect outputs. Review the final work — not a transcript, but actual completed tasks.

## Tone and Voice Guidelines

| Principle | Direction |
|---|---|
| **Confident, never arrogant** | State what Batonbot does without hype or hyperbole |
| **Developer-to-developer** | Write like you've shipped code. Assume technical literacy. |
| **Action-oriented** | Use verbs: Describe. Arrange. Execute. Review. Complete. Deliver. |
| **Outcome-driven** | Focus on what gets done, not how clever the tool is. Tasks completed, not tokens generated. |
| **Contrast-driven** | Frame the shift: Chat → Execution. Conversation → Completion. Chatbot → Agent. |
| **Minimal fluff** | No "revolutionary," "game-changing," or "next-gen." Let the results speak. |
| **User-controlled** | Emphasize agency. The user decides. The user arranges. The user reviews. |

## Messaging Philosophy

Batonbot is not a chatbot interface. It is an execution engine. Every piece of copy should reinforce that distinction:

- **Chatbots converse.** Batonbot completes.
- **Chatbots suggest.** Batonbot executes.
- **Chatbots require follow-up.** Batonbot delivers output.

The user comes to Batonbot with work to do — not curiosity to satisfy. Every headline, every CTA, every description should orient toward completion, control, and measurable results.
